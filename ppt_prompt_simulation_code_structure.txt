반도체 CMP 공정 APC 시스템 "Simulation 코드 구조" 교육용 PPT를 만들어줘.

[자료의 목적과 청중]
- 청중: Merge Data / 학습 파이프라인(Pre_Thk_VM·RR·Offset) 코드 구조 편을 이미 들었고,
  이제 Simulation 코드를 직접 읽고 수정해야 하는 사내 엔지니어
- 목적: simulation/{공정명}/Simulation_Hub.py에서 시작해 Common/Simulation.py의 run()이
  어떤 함수를 어떤 순서로 호출하는지 파악하고, "어떤 수정은 어느 함수만 보면 되는지"
  알 수 있게 하는 코드 워크스루
- 관통 메시지 2개 (자료 전체에서 반복 강조):
  ① Simulation은 학습 파이프라인의 "최종 소비자" — 새로 학습하지 않는다.
    Module이 저장한 학습 3종(Pre VM / RR / OFFSET) + Online 결과 + Ref lot을
    실측 데이터에 시점 기준으로 붙여서 시뮬레이션 값(RR_DB, Pre_Thk)을 재구성할 뿐
  ② simulate()는 조립 라인이다 — DataFrame 하나가 _build → _attach_* 5개 →
    보정 → 산식 → _finalize 를 순서대로 통과하며 컬럼이 붙는다.
    "한 단계 = 한 데이터 소스"라서 단계 순서만 외우면 전체가 읽힌다

[디자인 공통 지침]
- 깔끔한 기업 교육자료 스타일, "코드 지도" 느낌 (앞선 코드 구조 편들과 동일 톤)
- 함수명·파일명·컬렉션명은 전부 고정폭 폰트
- 호출 트리/시퀀스의 화살표에는 "왜 호출하는지" 짧은 라벨을 달 것
- 데이터(MongoDB 컬렉션·CSV)는 원통 도형, 함수/처리는 사각형으로 구분
- 각 파트 첫 장에는 전체 흐름 미니맵을 구석에 두고 현재 위치를 하이라이트

- 실행 순서 (순서가 중요한 이유를 화살표 라벨로):
  ① TIME(13P) 루프: FB_Type=='TIME' 행마다 simulate_time()
     → 결과에 Pre_Target_13P/Target_13P 부여 → zones['13P']에 누적
     → 이 루프에서 Thk_Para_13P 확보               ← PRESSURE가 기준으로 쓸 값
  ② PRESSURE 루프: FB_Type=='PRESSURE' 행마다
     · Thk_Para_13P 미확보면 전체 skip              ← 방어: 13P 없이 존 시뮬 불가

  (source 코드의 eval/exec 동적 변수명 방식을 defaultdict로 대체한 지점)

- simulate_time / simulate_pressure는 mode('TIME'/'PRESSURE')만 다른 얇은 래퍼
  → 실체는 simulate() 하나 (TIME/PRESSURE 공통 코어, mode 인자로 분기 흡수)

[조립 라인 9단계 — DataFrame이 왼쪽에서 오른쪽으로 흐르는 컨베이어로 그려줘.
 각 단계 위에 "무엇이 붙는가", 아래에 "join 방식"을 라벨로]
  ① _build_base_frame()      : 실측에서 Oper/Recipe/APC 필터 + 필요 컬럼만 추출
                               (OFFSET 결측→0, substrate_id dedup, Date 정렬)
  ② _attach_pre_vm()         : + Pre_Thk 학습값        [merge_asof, pre_eq_ch별]
  ③ _attach_rr()             : + RR 계수 4종(b1/b0)    [merge_asof, eqp+recipe별]
  ④ _attach_online()         : + Online Simulation 결과 [merge, substrate_id]
  ⑤ _attach_offset()         : + Simul_OFFSET          [merge_asof, eqp+recipe+IDLE별]
                               (TIME 모드는 LC/TB/Normal 분리 후 결합 —
                                OFFSET 학습의 저장 형식과 맞추기 위함)
  ⑥ _fill_learning_values()  : 결측 학습값 보정 사다리 (PART 3)
  ⑦ _apply_pre_thk_formula() : PRE_THK_FORMULA대로 Pre_Thk 조합 (PART 3)
                               (_resolve_pre_thk_formula가 미지정 항목을
                                Set-up 기준 기본값으로 확정)
  ⑧ _attach_ref_lots()       : + Ref wafer 1~4의 Post/Pre_VM/APC/OFFSET 값
                               [item_value의 ';' 목록을 Ref_i 컬럼으로 분해 후 merge]
  ⑨ _finalize()              : 출력 컬럼 표준화(Pol_Time/THK 등) + RR_DB 산출 (PART 3)

PART 3. 조립 라인의 핵심 부품 3개 상세

[부품 1: _fill_learning_values() — 결측 보정 사다리]
- 학습값이 비어 있는 행을 단계적으로 채우는 우선순위 사다리 (표로):
  | 대상 | 1차 | 2차 | 3차 |
  | RR b1/b0 | eq_model_recipe별 ffill | eq_model_recipe별 평균 | — |
  | Simul_OFFSET | eq_model_recipe+IDLE별 평균 | (이후 0) | — |
  | Pre_Thk | pre_eq_ch별 ffill | pre_eq_ch별 평균 | 0 |
- RR_ 접두 컬럼은 to_numeric(coerce) 선행 — '-' sentinel(과거 문서 호환)을 NaN으로

[부품 2: Pre_Thk 산식 — runner의 PRE_THK_FORMULA가 코드까지 내려오는 경로]
- _resolve_pre_thk_formula(sk, formula): 산식 확정 규칙
  · runner가 지정한 항목은 그대로, 미지정 항목은 Set-up 기준 기본값
    (Pre_Oper_Code{n} 있으면 'reg', SOURCE OX CMP의 OPER2만 weight 2 — source 동작 보존)
  · 'raw'/'reg' 문자열은 (mode, 1) 튜플로 정규화
- _apply_pre_thk_formula(df, sk, resolved): 확정 산식대로 조합
  Pre_Thk = (PRE_OPER=True면 moving avg 학습값, False면 0)
            + Σ weight × (raw: 계측값 | reg: 계측값×b1+b0)
  · 적용 전 원값은 Pre_Thk2로 보존 (검증용)
  · 계측 컬럼/회귀계수 없으면 해당 항만 제외하고 진행 (경고 출력)
- 포인트: 산식 실험은 runner의 PRE_THK_FORMULA만 바꾸면 됨 — 라이브러리 수정 불필요

[부품 3: _finalize()의 RR_DB — 학습된 4개 모델 중 하나를 고르는 우선순위]
- RR 편에서 배운 4개 모델(if/current/weighted/기본)이 여기서 소비됨:
  | 순위 | 조건 | 사용 계수 |
  | 1 | if_b1 있음 + 소모품 ≤ Pad_Seperation | RR_if_b1/b0 (패드 초기 구간) |
  | 2 | current 있음 | RR_b1/b0_current (현재 사이클) |
  | 3 | weighted 있음 | RR_b1/b0_weighted (최근 가중) |
  | 4 | (항상) | RR_b1/b0 (기본) |
  → 학습 편의 "성립된 모델만 저장" 정책과 짝: 저장 안 된 모델은 NaN이라 자동으로
    다음 순위로 넘어감 (np.vectorize로 행별 적용)

[각 항목을 문제→증상→해결 3단으로]
① wafer 단위 join 개선 — merge_pre_oper_info()
   · 문제: source 코드는 PRE_THK_INFO를 alias_lot_id(lot 단위)로 결합
   · 증상: wafer별 계측 차이가 lot 첫 행 값으로 뭉개짐
   · 해결: substrate_id(wafer 단위) join으로 변경 — 학습측
     apply_pre_oper2_correction과 동일 기준 (학습·시뮬 일관성)
② 구/신 스키마 공존 대응: coalesce_substrate_id() + drop_duplicates(keep='last')
   + '-' → NaN 복원 + merge_df와 겹치는 컬럼 제거 (suffix 충돌 방지)
③ 회귀전용 컬렉션의 reindex — _attach_pre_vm()
   · 회귀계수만 저장된 컬렉션은 pre_eq_ch/Pre_Thk/Count 키 자체가 없음
   · reindex(columns=...)로 NaN 컬럼을 강제 생성 → 후속 코드 분기 통일
④ eval/exec 제거 — _append_zone() + defaultdict
   · source는 zone별 결과를 eval/exec 동적 변수명으로 누적 → dict 누적으로 대체
   · zone 추가가 extra_zones 인자 등록만으로 가능해진 이유
⑤ TIME 선행 방어: PRESSURE는 Thk_Para_13P가 있어야 실행
   → TIME set-up 없으면 명시적 skip 메시지 후 중단 (조용한 오답 방지)
⑥ 남겨둔 확인 사항 (수정 아님, 인지용):
   · Ref_YN == Ref_Count (source 원본 그대로 — Y/N 의도 여부 회사 확인 예정)
   · CSV 출력은 Spotfire 연동용 현행 유지 — 추후 DB 적재/웹 뷰어로 대체 예정
